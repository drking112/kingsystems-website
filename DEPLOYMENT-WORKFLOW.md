## Recommended Production Workflow

### Option 1: GitHub + Cloudflare Pages (RECOMMENDED - FREE)

**Benefits:**
- ✅ Free forever
- ✅ Automatic deployments on git push
- ✅ Version control for all changes
- ✅ Professional workflow
- ✅ No file size limits for media
- ✅ Easy rollback if something breaks

**Setup Steps:**

1. **Create GitHub Repository**
   - Go to: https://github.com/new
   - Name: kingsystems-website
   - Public repository
   - Don't initialize

2. **Upload Files to GitHub**
   - Download all project files as ZIP
   - Extract locally
   - Use GitHub Desktop or command line:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/YOUR-USERNAME/kingsystems-website.git
   git push -u origin main
   ```

3. **Connect Cloudflare Pages to GitHub**
   - Cloudflare Dashboard → Pages
   - "Connect to Git"
   - Select your repository
   - Build settings: (leave empty for static site)
   - Deploy

4. **Every Update:**
   - Edit files locally
   - Commit and push to GitHub
   - Cloudflare auto-deploys (30 seconds)

### Option 2: Manual ZIP Upload (Current Method)

## Media Storage Solutions

### For Logo & Small Assets (< 1MB each)
**Store in GitHub repository:**
- Add to `images/logo/` folder
- Deploys automatically with site
- Reference: `<img src="/images/logo/logo.png">`

### For Large Media (Photos, Videos)
**Option A: ImgBB (Free, Recommended)**
- URL: https://imgbb.com
- Upload images
- Copy direct link
- Use in HTML

**Option B: GitHub LFS (Large File Storage)**
- For files > 100MB
- Stores large media in GitHub
- Free tier: 1GB storage

**Option C: Cloudflare R2 (Free tier: 10GB)**
- Similar to AWS S3
- Free tier available
- Better than Cloudflare Images for staging
